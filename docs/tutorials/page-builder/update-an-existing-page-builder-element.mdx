## Overview

All Page Builder elements are plugins which means we can easily modify them or even add new ones.
Let's see how we can add two predefined settings that is padding and width in the button element.

## Plugin Type

There's only one plugin you need to modify to update an element: [`PbEditorPageElementPlugin`](/docs/references/page-builder/plugins#pb-editor-page-element).
Plugins of this type require a create function (to create an element data structure), and a render function that returns a React element that will be rendered in the editor.

All element plugins are registered as factory so you can pass the following options to them:

```ts title="@webiny/app-page-builder/types"
type PbEditorElementPluginArgs = {
  // A function to create an element data structure.
  create?: (defaultValue) => Record<string, any>;
  // Array of element settings plugin names.
  settings?: (defaultValue) => Array<string | Array<string | any>>;
  // A function that return the toolbar properties of the element plugin.
  toolbar?: (defaultValue) => Record<string, any>;
  elementType?: string;
};
```

## Add Settings in an Element

Please head over to `apps/admin/code/src/plugins/pageBuilder.ts` where all the plugins related to the Page Builder app are located.

As you can see there are two core sets of plugins:

1. Editor plugins: needed for Page Builder editor
2. Render plugins: needed for Page Builder page preview and to render the actual page in the Website app

For this tutorial, we only need to update the editor plugin.

### Add Settings
Now we're going to add the padding and width settings to the button element.
Let us make the following change:

```ts title="apps/admin/code/src/plugins/pageBuilder/editorPlugins.ts:92"

button({
  settings: defaultValue => [
    ...defaultValue,
    "pb-editor-page-element-style-settings-padding",
    "pb-editor-page-element-style-settings-width"
  ]
})

//
// Other element plugins.
//
```

### Update Stylesheet

To fix this, we need to update the button element style within the theme
so that it takes those settings applied via the editor.

Edit the `.webiny-pb-page-element-button` class inside `apps/theme/pageBuilder/styles/elements/button.scss`
and make the following changes:
- remove `padding` property
- update `width` property

```scss title="apps/theme/pageBuilder/styles/elements/button.scss"
// Some code is removed for the sake of brevity.

.webiny-pb-page-element-button {
  // For the "width" settings in editor sidebar to work
  // The element needs to have "width: 100%"
  width: 100%;
}

//
// More style definition.
//
```

After saving these changes the previously applied changes will show in the editor.

We can easily fix this by adding the default values for both `padding` and `width` properties.

Here is how to do it:

```ts {9-33} title="apps/admin/code/src/plugins/pageBuilder/editorPlugins.ts"
// Some code is removed for the sake of brevity.

button({
  settings: defaultValue => [
      ...defaultValue,
      "pb-editor-page-element-style-settings-padding",
      "pb-editor-page-element-style-settings-width"
  ],
  create: defaultValue => {
      const elementData = { ...defaultValue };

      // Set default value for the padding property
      set(
          elementData,
          "data.settings.padding",
          createInitialPerDeviceSettingValue(
              { advanced: true, top: "12px", right: "16px", bottom: "12px", left: "16px" },
              DisplayMode.DESKTOP
          )
      );

      // Set default value for the width property
      set(
          elementData,
          "data.settings.width",
          createInitialPerDeviceSettingValue(
              { value: "150px" },
              DisplayMode.DESKTOP
          )
      );

      return elementData;
  }
})

//
// Other element plugins.
//
```

:::info
The `create` function defines the initial value for element `settings` and `data` property which can hold any data you might need.
:::

After saving these changes, if you try dropping a new button element to the page it looks much better thanks to the default `padding` and `width` as shown below:
